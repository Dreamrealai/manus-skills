# Browser Automation Reference

Detailed browser guidance for launching Gemini and ChatGPT Deep Research, preserving the correct ChatGPT thread, and harvesting the completed reports without falsely treating the session as failed.

## Selector Volatility Warning

ChatGPT and Gemini frequently change their UI. Use the selectors and anchors below as guidance, but fall back to visible text matching when needed.

| Surface | Preferred anchor | Fallback |
|---|---|---|
| ChatGPT prompt input | `id="prompt-textarea"` | The primary chat composer textarea |
| ChatGPT send button | `[data-testid="send-button"]` when present | Visible send icon/button near the composer |
| Gemini input | Main textarea in the chat input area | Visible primary prompt box |
| Plan confirmation | Visible button text such as `Start research`, `Confirm`, or equivalent | Use nearby visible action button after the plan loads |
| ChatGPT report controls | Visible expand, open, or download icon in the report card chrome | Click the central report area first, then use visible controls |

## Gemini Deep Research Flow (`gemini.google.com/app`)

### Activation and Submission

1. Navigate to `https://gemini.google.com/app`.
2. Click **Tools** in the input area.
3. Select **Deep Research**.
4. Confirm the deep-research mode is active.
5. Enter the prompt and submit.
6. Review the research plan and click the confirmation button.
7. Leave the tab open in the background.

### Gemini Monitoring and Harvest

Check Gemini on the same 5-minute cadence used for ChatGPT. When the Gemini report is complete, copy the full report from the page and save it to the standardized raw `Gemini` filename generated by the helper script.

## ChatGPT Deep Research Flow (`chatgpt.com/deep-research`)

### Activation and Submission

1. Navigate directly to `https://chatgpt.com/deep-research`.
2. Enter the prompt in the textarea with `id="prompt-textarea"`.
3. Submit the prompt.
4. Review the plan and click the confirmation button.
5. **As soon as a specific `https://chatgpt.com/c/<conversation_id>` URL exists, save it.** This direct thread URL is the primary recovery path later.
6. Start the local tracker so the 5-minute monitoring cadence is documented.
7. Start the reminder sidecar so the next due check is continuously surfaced while other work continues.

### Monitoring Notes

| Situation | Correct interpretation | Required action |
|---|---|---|
| Generic home view shows only the composer | The report may still exist in the saved thread | Reopen the direct `/c/<conversation_id>` URL |
| Thread appears visually blank | The report may be outside the current viewport | Use **Scroll to bottom** |
| Only the prompt is extractable | The report UI has not been surfaced yet | Jump to bottom, then try the extraction methods in order |
| ChatGPT is still running after 20 minutes | Slow completion, not automatic failure | Keep 5-minute checks active while drafting other materials |
| Direct thread URL was never saved | Recovery is still possible | Open the most recent matching conversation from the sidebar and save its direct URL immediately |

## ChatGPT Harvest — Primary Method Plus Two Fallbacks

Use these methods in order before assuming the ChatGPT report is missing.

### Method 1: Central Report-Area Expansion

This is the primary working method and should be tried first because the report is often nested inside a central card.

1. Open the saved `https://chatgpt.com/c/<conversation_id>` thread.
2. If the page looks blank or only shows the composer, keep the sidebar visible and confirm the conversation title.
3. Use the visible **Scroll to bottom** control.
4. Click directly into the **central report area** so the nested report expands into a more readable state.
5. Wait for the report body to render fully.
6. Use the visible copy affordance or manual extraction only after the full body is visible.
7. Save the content to the standardized raw `ChatGPT` filename.

### Method 2: Expand or Download Controls

If the central-area click does not work, try the controls in the report card chrome.

1. Look for a visible **expand**, **open**, or **download** icon.
2. Click the icon that produces the most readable or directly exportable version of the report.
3. If a download/export works, save that artifact and keep the copied text alongside it when practical.
4. Save the harvested report with the standardized raw `ChatGPT` filename.

### Method 3: Flattening Prompt Recovery

If the report remains nested, partially hidden, or uncopyable, use the model-switch fallback.

1. Stay in the same thread.
2. Switch the model to **GPT 5.4 Thinking**.
3. Send this exact recovery prompt:

> Please give the response back to me without nesting it or changing the content.

4. Wait for the flat restatement.
5. Save that flat version as the harvested ChatGPT output, and note in the working log that the flattening fallback was used.

## If the Direct Thread URL Was Missed

1. Go to `https://chatgpt.com/`.
2. Look in the sidebar for the most recent conversation whose title matches the research prompt.
3. Open it and confirm the content matches the in-flight deep-research task.
4. Save the resulting direct `/c/<conversation_id>` URL immediately.
5. Resume the normal direct-thread recovery and harvest flow.

## What Not to Do

| Method | Why it fails or misleads |
|---|---|
| Treating the generic home view as the source of truth | It often hides the real report thread |
| Assuming a blank view means the job failed | The thread may simply need to be reopened and scrolled |
| Extracting before the report is visibly loaded | You may capture only chrome, prompt text, or an empty shell |
| `Ctrl+A` / Select All | Usually grabs page chrome rather than the report |
| Backend scraping | Blocked and unreliable |

## Final-Check Rule

Before any final deliverable is sent to the user, return to the saved ChatGPT thread one last time. ChatGPT Deep Research must not be silently skipped, even if the workflow already drafted interim synthesis after the 20-minute mark.
